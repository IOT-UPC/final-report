# Capítulo III: Requirements Specifications

## 3.1. User Stories

A continuación se presenta el conjunto de Epics y User Stories definidas para el producto PSYMED. Cada Epic agrupa un conjunto de funcionalidades relacionadas, y cada User Story describe un requisito desde la perspectiva del usuario, acompañada de sus criterios de aceptación redactados en formato Gherkin (Given-When-Then), en tiempo presente, tercera persona y sin referencia a detalles de interfaz de usuario. Se incluyen asimismo Technical Stories para las funcionalidades de la RESTful API que no tienen interacción directa con los usuarios finales.
